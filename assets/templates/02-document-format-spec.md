# 研发文档格式规范
## 一、通用规范
### 1.1 文件命名
- 格式：英文+下划线，禁止空格与特殊字符
- 示例：`PRD_V1.0.0.md`、`SRS_V1.0.0.md`、`HLD_V1.0.0.md`

### 1.2 文档结构
- 固定结构：基础信息 → 变更履历 → 核心内容 → 附件

### 1.3 标题层级
- 一级：文档标题
- 二级：大章节
- 三级：子章节
- 四级：细项
- 禁止跨级

## 二、Markdown规范
- 代码块必须声明语言类型
- Mermaid 图必须可编辑、可追溯，禁止截图替代
- 链接优先相对路径

## 三、版本与变更
- 版本号采用 `V主.次.补丁`
- 变更履历必须包含：版本、日期、更新人、核心变更、关联记录

## 四、质量要求
- 术语统一、描述无歧义
- 所有规则可验证、可落地
- 敏感信息必须脱敏

## 五、工程一致性与审计门禁（强制）
### 5.1 源码一致性
- 接口路径、响应结构、配置项名称必须与源码一致
- 核心流程图必须映射真实模块

### 5.2 追溯门禁
- PRD→SRS→HLD/LLD 需完整映射
- LLD 必须有代码落点与验证方式

### 5.3 发布门禁
- 目录分层完整（01-prd/02-srs/03-hld/04-lld）
- 文档索引完整（README）
- 追溯矩阵完整（TRACEABILITY）
- 出版清单完整（PUBLISH_CHECKLIST）
- 评审清单完整（REVIEW_CHECKLIST）
